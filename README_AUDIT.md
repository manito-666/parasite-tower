## 💡 系统设计理解

### 濒死转移流程 (HP=0 时)

```
HP <= 0
   ↓
[是否有备用形态?]
   ├─ 否 → triggerDeath() (游戏结束)
   └─ 是 → 意识抽离
         ↓
      checkDeathTransfer()
         ↓
      [存活形态数量?]
      ├─ 1 个 → 自动转移
      └─ 多个 → showTransferOverlay() (让玩家选择)
         ↓
      executeDeathTransfer(selectedForm)
         ↓
      loadForm() → 加载新形态
         ↓
      _stiffnessTurns = 1 (1 回合无法行动)
         ↓
      检查孤狼模式 (仅1形态存活)
```

### 关键状态变量

| 变量 | 类型 | 用途 |
|------|------|------|
| `_deadForms` | boolean[] | 标记各槽位是否死亡 |
| `_loneWolf` | boolean | 孤狼模式 (仅1形态) |
| `_stiffnessTurns` | number | 僵直回合数 |
| `_inCombat` | boolean | 战斗状态锁定 |
| `target` | monster | 当前战斗目标 |

## 📞 故障排除

### 问题: 死亡形态没有灰化效果
**原因**: CSS 选择器不匹配 (Bug #1)
**解决**: 修改 `.form-slot.dead-form` 为 `.fslot.dead-form`

### 问题: 转移后还能看到怪物
**原因**: 没有调用 `closeCombat()` (Bug #4)
**解决**: 在转移前添加 `closeCombat()`

### 问题: 紧急接管消息重复
**原因**: 消息在两个地方重复添加 (Bug #2)
**解决**: 移除 onclick 处理中的重复代码

### 问题: 可以快速连击绕过僵直
**原因**: 缺乏战斗状态锁定 (Bug #5)
**解决**: 添加 `_inCombat` 锁定检查

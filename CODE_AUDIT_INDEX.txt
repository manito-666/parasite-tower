🚨 最关键的 4 个修复
════════════════════════════════════════════════════════════════════════════════

1. CSS 选择器不匹配 (行 681-687)
   问题: .form-slot.dead-form → .fslot.dead-form
   影响: 死亡形态无法显示灰化和骷髅标记

2. 战斗状态未清理 (行 2640-2648, 4458-4467)
   问题: 转移时缺少 closeCombat() 调用
   影响: 战斗状态不一致，游戏可能卡住

3. 战斗锁定机制 (行 2463)
   问题: 缺乏 _inCombat 状态锁定
   影响: 快速连击可绕过僵直保护

4. 紧急接管代码重复 (行 4782-4821)
   问题: 消息和状态重置在多个地方重复
   影响: 游戏流程混乱，UX 差


💡 核心概念
════════════════════════════════════════════════════════════════════════════════

【濒死转移流程】
  HP <= 0 → 意识抽离 → 选择形态 → 转移 → 重组僵直(1回合) → 恢复

【关键状态变量】
  • _deadForms[]      - 各槽位死亡标记
  • _loneWolf         - 孤狼模式 (仅1形态)
  • _stiffnessTurns   - 僵直回合数
  • _inCombat         - 战斗锁定状态 (新增)
  • target           - 当前战斗目标

【故障场景】
  ❌ UI 缺失:        死亡形态无法灰化 (Bug #1)
  ❌ 卡住:           战斗状态混乱 (Bug #4, #13)
  ❌ 绕过保护:       无法阻止快速连击 (Bug #5)
  ❌ 混乱流程:       重复消息 (Bug #2)
